<b>pgmnemo v0.9.0</b> — the token-economy release ⚡

This release makes <code>navigate_locate</code> actually cheap, scopes it to a project, fixes hybrid recall on real-size corpora, and lays the schema groundwork for typed/multimodal content.

<b>Fixed — the moat that wasn't shipping:</b>
• <b>navigate_locate budget now counts what it returns.</b> It was charging the full lesson text against your token budget while only returning a 50-char preview — so a 2000-char budget gave you ~8 candidate IDs instead of ~40. Locate is now genuinely cheap: rank, hand back many lightweight IDs within budget, expand only what you pick. ⚠️ <b>Behavioral change:</b> the same <code>token_budget_chars</code> now yields ~5× more IDs — lower your budget proportionally to keep prior result counts.

<b>Added:</b>
• <b><code>project_id_filter</code> on navigate_locate</b> — scope locate to one project/namespace (parity with <code>recall_hybrid</code>; backward-compatible, optional 5th arg).
• <b>Typed-content columns</b> — <code>content_type</code>, <code>blob_ref</code>, <code>doc_ref</code> (all nullable, additive) — foundation for typed and multimodal memory.
• <b>Selective embedding</b> — a precise fact can be ingested with a NULL embedding (indexed by keyword/btree) without being excluded from recall.

<b>Changed — recall_hybrid is fast on real corpora:</b>
• Rewrote <code>recall_hybrid</code> as two bounded retrieval arms (vector + keyword) fused by RRF, so HNSW is actually used instead of scanning every row. On a 2500-row corpus: identical top-10 (Jaccard 1.00, recall@10 78.8% vs 78.1%) at lower latency; on production-scale corpora it ends the multi-second full-scan that previously timed out.
