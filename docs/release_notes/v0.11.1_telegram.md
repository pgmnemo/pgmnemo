# pgmnemo v0.11.1 — типизированный поиск на горячем пути

**pgmnemo v0.11.1** — патч-релиз, завершающий поддержку типизированного поиска.

В v0.11.0 параметр `p_content_types` появился в `recall_hybrid()`, но агенты и MCP-инструмент `pgmnemo.recall` вызывают именно `recall_fast()` — таким образом, фильтрация по типу контента оставалась недоступна на практике. Этот выпуск исправляет ситуацию.

---

**Что изменилось:**

- `recall_fast()` — новый 6-й параметр `p_content_types TEXT[] DEFAULT NULL`
- `recall_lessons()` — новый 8-й параметр `p_content_types TEXT[] DEFAULT NULL`
- Семантика идентична `recall_hybrid()`: `NULL` → все типы (без изменений), массив → только строки с `content_type = ANY(...)`, `'{}'` → ноль строк

**Обратная совместимость:** все существующие вызовы работают без изменений — параметр `DEFAULT NULL`.

---

**Как использовать:**

```sql
-- только уроки типа "procedure"
SELECT * FROM pgmnemo.recall_fast(
    query_embedding,
    k             => 10,
    role_filter   => 'software_developer',
    p_content_types => ARRAY['procedure']
);

-- только факты и паттерны
SELECT * FROM pgmnemo.recall_lessons(
    query_embedding, 10, 'software_developer',
    NULL, NULL, NULL, NULL,
    ARRAY['fact', 'pattern']
);
```

---

**Обновление:** `ALTER EXTENSION pgmnemo UPDATE TO '0.11.1';`
