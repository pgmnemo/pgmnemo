<b>pgmnemo v0.11.1</b> — типизированный recall теперь и на горячем пути (recall_fast)

В v0.11.0 фильтр по типу контента появился в <code>recall_hybrid()</code>. В этом патче он добавлен в функции, которые реально используются в горячем пути: <code>recall_fast()</code> и <code>recall_lessons()</code> — именно их вызывает MCP-инструмент <code>_search_lessons</code> в Agency.

<b>Что изменилось</b>

<code>recall_fast()</code> — новый необязательный 6-й параметр <code>p_content_types TEXT[] DEFAULT NULL</code>:
<pre>-- как раньше: вернёт всё
SELECT * FROM recall_fast(embedding, 10, 'technical_lead');

-- новое: только нужные типы
SELECT * FROM recall_fast(embedding, 10, 'technical_lead', NULL, NULL,
                          ARRAY['procedure', 'fact']);</pre>

<code>recall_lessons()</code> — новый необязательный 8-й параметр <code>p_content_types TEXT[] DEFAULT NULL</code>. На гибридном пути фильтр пробрасывается в <code>recall_hybrid()</code>; на векторном пути — применяется напрямую в WHERE.

<b>Полностью обратно совместимо</b>

• Новые параметры стоят последними, со значением по умолчанию NULL.
• Все старые вызовы работают без изменений.
• NULL — поведение в точности прежнее (все типы контента).
• Пустой массив <code>ARRAY[]::text[]</code> — явно возвращает ноль строк.

<b>Честно о том, что это</b>

Это фильтр на горячем пути — не измеренное улучшение качества recall. Формула ранжирования не менялась. Версия v0.11.0 принесла типизированный recall в <code>recall_hybrid()</code>; v0.11.1 делает его физически доступным там, где его на самом деле используют потребители.

<b>Обновление</b>

<pre>ALTER EXTENSION pgmnemo UPDATE TO '0.11.1';</pre>

MCP-сервер (если используется):
<pre>pip install --upgrade pgmnemo-mcp==0.11.1</pre>
