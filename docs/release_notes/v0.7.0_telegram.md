<b>pgmnemo v0.7.0</b> — Outcome-Learning Loop 🎯

Agents can now teach pgmnemo from experience. Every lesson has a <b>confidence score</b> [0,1] that tracks its outcome history — updated by calling <code>reinforce()</code> after each agent run.

<b>What's new:</b>

• <code>confidence REAL DEFAULT 0.5</code> column on <code>agent_lesson</code> — tracks per-lesson outcome track record
• <code>pgmnemo.reinforce(lesson_id, outcome)</code> — adjust confidence from agent feedback:
  — <code>'success'</code> → +0.10 (clamped to 1.0)
  — <code>'failure'</code> → −0.15 (clamped to 0.0)
  — <code>'neutral'</code> → no change
• <code>recall_hybrid()</code> gains <b>two new output columns</b>: <code>confidence</code> and <code>match_confidence</code> (interpretable [0,1] quality indicator)
• <b>Confidence-aware ranking</b> — high-confidence lessons rank above low-confidence ones in tie-break situations (auxiliary term: <code>+0.025 × confidence</code>)
• <b>ingest() quality guards</b> — rejects lessons shorter than 20 chars or with >80% single-token repetition
• <b>stats() distribution</b> — new columns: <code>confidence_mean</code>, <code>confidence_p10/p50/p90</code>, <code>confidence_below_threshold_count</code>

<b>Bug fixed:</b>
<code>graph_proximity</code> CTE alias bug in <code>recall_hybrid()</code> that caused <code>ERROR: missing FROM-clause entry for table "gw"</code> during graph traversal. Fix is in both fresh-install and upgrade scripts.

<b>Upgrade:</b>
<pre>ALTER EXTENSION pgmnemo UPDATE TO '0.7.0';</pre>
Adds columns with defaults — no data loss, brief lock only.
