<b>pgmnemo v0.10.0</b> — быстрый recall для интерактивных запросов + гайд по интеграции с агентом

<b>recall_fast() — отдельный быстрый путь</b>

Когда агент спрашивает память прямо во время работы (а не один раз на старте), ответ нужен мгновенно. <code>recall_fast()</code> — это HNSW-поиск без графовой части и без записи статистики recall, цель — отклик за миллисекунды. Полный <code>recall_hybrid()</code> остаётся для офлайн-сценариев, где качество важнее скорости.

<b>AGENT_INTEGRATION.md — как подключить память к агенту</b>

Когда дёргать recall по ходу задачи, как скоупить по роли / проекту / запуску, какие инструменты давать агенту (<code>recall</code> + <code>patch</code>, не <code>ingest</code>), и почему recall — дополнение к стартовому контексту, а не замена.

<b>Гайд по confidence-ранжированию</b>

<code>reinforce()</code> и <code>pgmnemo.confidence_boost_weight</code> есть с v0.7, но по умолчанию выключены — и многие пропускают главное: ранжировать уроки по тому, что реально срабатывало, а не по свежести. Гайд показывает, как включить и настроить.

<b>Обновление</b>

Расширение PostgreSQL:
<pre>ALTER EXTENSION pgmnemo UPDATE TO '0.10.0';</pre>

MCP-сервер:
<pre>pip install --upgrade pgmnemo-mcp</pre>
