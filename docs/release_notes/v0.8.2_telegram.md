<b>pgmnemo v0.8.2</b> — adopter fixes + MCP that just works 🔌

A patch driven by real production reports. No schema change — <code>ALTER EXTENSION pgmnemo UPDATE TO '0.8.2'</code>.

<b>Fixed (the "my recall is empty" class):</b>
• <b>Consistent <code>include_unverified</code> parsing</b> — every recall path now accepts <code>on</code>/<code>true</code>/<code>1</code> (one function only honoured the literal <code>'on'</code>).
• <b>No more silent empty recall.</b> If recall returns nothing while unprovenanced "ghost" lessons match, pgmnemo now emits a <code>NOTICE</code> telling you exactly why and how to include them — instead of leaving you guessing.
• Docs: <code>ALTER DATABASE SET</code> applies only to new connections (restart the MCP/pool).

<b>Added — the MCP is now drop-in:</b>
• <b>Self-embedding via <code>EMBEDDING_SERVER</code></b> — point the MCP at any OpenAI-compatible embeddings endpoint and it embeds queries/lessons itself → real vector+BM25 hybrid recall, no out-of-band vectors. Unset → text-only fallback.
  <pre>"env": { "DATABASE_URL": "...", "EMBEDDING_SERVER": "http://host:1234/v1/embeddings" }</pre>
• <b>Docker image</b> — run the MCP in a container so its deps don't fight your agent libraries on Linux:
  <pre>docker pull gaidabura/pgmnemo-mcp:0.8.2</pre>
  Multi-arch (amd64/arm64), auto-built and pushed to Docker Hub on every release tag.

<b>pg_regress:</b> 22/22 ✅ · MCP ingest+recall verified end-to-end inside the published image.
