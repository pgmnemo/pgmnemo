<b>pgmnemo v0.8.3</b> — docs patch 📝

We ran a cold first-setup test (a fresh engineer adopting pgmnemo with zero insider knowledge) and fixed exactly what tripped them. No schema or scoring change — SQL is byte-identical to v0.8.2.

<b>Fixed:</b>
• <b>The "Verify install" smoke SQL was broken</b> — the very first command a new user runs. It failed twice (the bare <code>NULL</code> couldn't resolve the <code>ingest()</code> overload, and the example lesson was under the 20-char minimum). Now correct: <code>NULL::vector(1024)</code>, <code>3::smallint</code>, valid-length text.
• <b>MCP tool-argument contract was mis-documented</b> — the README implied <code>ingest</code> took a nested <code>metadata</code> dict, but the real schema exposes <code>text</code>/<code>role</code>/<code>topic</code>/<code>importance</code>/<code>project_id</code>/<code>commit_sha</code>/<code>artifact_hash</code>/<code>metadata</code> as <b>top-level</b> args. An agent following the old docs would silently mis-scope its lessons. Fixed + defaults documented.
• <b>Version drift</b> in install examples (pinned <code>v0.8.1</code>) → current, plus a <b>from-zero MCP quickstart</b> (fresh Postgres → extension → MCP container → verify).

Upgrade is a no-op at the SQL level:
<pre>ALTER EXTENSION pgmnemo UPDATE TO '0.8.3'</pre>
