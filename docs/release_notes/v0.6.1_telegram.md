🚀 <b>pgmnemo v0.6.1</b>

Два новых функционала: bitemporal фильтрация + стресс-тест при 10M строках.
RRF Fix-A (F1) отложен на v0.6.2 — real-DB бенч выявил регрессию.

<b>1. as_of_ts на recall_lessons() (F2)</b>
<code>recall_lessons(embedding, k, role, project_id, query_text, as_of_ts)</code>
6-й параметр <code>TIMESTAMPTZ DEFAULT NULL</code>. Когда задан: возвращает только уроки где <code>t_valid_from ≤ as_of_ts &lt; t_valid_to</code>.

Используется для DAG-агентов (bitemporal consistency между фазами RESEARCH→PLAN→IMPLEMENT), точечного воспроизведения состояния памяти, аудита «что агент знал в момент X».

Передаётся в <code>recall_hybrid()</code> через транзакционный GUC <code>pgmnemo.as_of_timestamp</code>. Backward-compatible: существующие вызовы без <code>as_of_ts</code> не меняют поведения.

<b>2. Стресс-тест recall_lessons() при 100K/1M/10M строках (issue #29)</b>
<code>benchmarks/scripts/stress_recall_large.py</code>
Синтетический корпус, without embeddings (BM25 path), P50/P95/P99 латентность.
Цели: 100K → P99 ≤ 500ms; 1M → P99 ≤ 2000ms; 10M → P99 ≤ 8000ms.
pg_regress: <code>stress_recall</code> — HNSW + GIN индексы, scale targets, ef_search GUC.

<b>RRF Fix-A (F1) — отложен на v0.6.2</b>
Real-DB бенч (LME-S, 500 вопросов, bge-m3 1024d) показал регрессию −22.44pp recall@10 при ORDER BY rrf_diag с фиксированным k=60 на смешанных BM25/не-BM25 корпусах.
Формула ранжирования в v0.6.1 <b>не изменилась</b> от v0.5.1/v0.6.0 (ORDER BY fusion_score).
Детали и план фикса: <code>benchmarks/gate/v0.6.1.json</code>, CHANGELOG [0.6.1].
