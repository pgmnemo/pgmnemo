<b>pgmnemo v0.9.3</b> — saner reinforce() defaults + opt-in confidence ranking 🎛️

Two tuning releases bundled together. Both ship <b>off by default</b> — byte-identical to v0.9.1 unless you opt in.

<b>D1 — Base-rate-adjusted reinforce() deltas (on by default)</b>

The previous success delta (+0.10) caused confidence to saturate toward the ceiling at typical usage rates, making it stop discriminating between good and bad lessons. The new defaults are proportional:

• <b>Success delta:</b> +0.10 → +0.02
• <b>Failure delta:</b> −0.15 → −0.12

Both overridable per-session or at DB/role level:
<pre>SET pgmnemo.reinforce_success_delta = '0.05';
SET pgmnemo.reinforce_fail_delta = '0.08';</pre>

Clamp: [0.001, 0.5]. Applies to both scalar and batch <code>reinforce()</code>.

<b>I1 — Confidence-weighted recall ranking (opt-in GUC)</b>

<code>reinforce()</code> updates confidence per lesson, but the prior <code>recall_hybrid</code> score gave it negligible weight (~3 RRF positions maximum). This release adds an explicit tie-breaker term — off by default:

<pre>SET pgmnemo.confidence_boost_weight = '0.003';</pre>

At <code>w=0.003</code>, a high-confidence lesson gains ~8–15 RRF positions over a low-confidence one. Cold-start (confidence=0.5) gets zero boost or penalty. Clamped [0.0, 0.01].

<b>Upgrade:</b>
<pre>ALTER EXTENSION pgmnemo UPDATE TO '0.9.3'</pre>

New reinforce defaults apply immediately. The ranking GUC requires an explicit SET — no behavior change otherwise.
