# pgmnemo 0.7.1 — Release Note

**🩹 Calibration patch — `match_confidence` now works as documented**

---

**What changed**

`recall_hybrid()` output column `match_confidence` was returning ~0.005 for
strong semantic hits (cosine=0.52). Root cause: the formula used
`final_score / 1.5` but `final_score` on the RRF path is ~0.008–0.05, not ~1.5.
Fixed: `match_confidence = vec_score` (cosine similarity, already [0,1]).

Recall ranking is **unchanged** — `ORDER BY final_score` is identical to 0.7.0.

**Also added**

- `reinforce(lesson_ids BIGINT[], outcome TEXT) → INT` — batch form. Skips
  missing IDs silently, returns count updated. One round-trip instead of N.

**Upgrade**

```sql
ALTER EXTENSION pgmnemo UPDATE TO '0.7.1';
```

No schema changes. Function-only patch.

**⚠️ If you threshold on `match_confidence`:** values are now cosine [0,1].
Old values were ~0.005 for good hits. Update any `> 0.01` guard to a real
cosine threshold (e.g. `> 0.3`).
