<b>pgmnemo v0.9.1</b> — graph navigation that actually walks 🧭

The navigation loop is <code>locate → see what it's connected to → expand only what you pick</code>. Step 2 was quietly broken: <code>navigate_expand</code> filtered connections by a coarse 4-value category instead of the real edge type, so most real edges were invisible — graph expansion returned <b>zero neighbors</b>. This release fixes the one load-bearing primitive.

<b>Fixed — step 2 of the navigation loop:</b>
• <b>Traverse by edge type, not category.</b> <code>navigate_expand</code>/<code>navigate_locate</code> now filter on <code>relation_type</code> (the actual discriminator). New optional arg <code>relation_types TEXT[]</code> — <code>NULL</code> walks every active edge. A semantic/entity edge that returned empty in 0.9.0 now returns the neighbor.
• <b>Bidirectional.</b> Expansion follows edges both ways — discover a cause from its effect, not just forward.
• <b>Permissive by default.</b> Weight threshold 0.7 → 0.5; the agent decides which connections to follow.
• <b>Live-edge handling.</b> Edges marked with an <code>'infinity'</code> validity sentinel are no longer dropped.
• <b>Faster locate.</b> Topic keyword match moved to a stored GIN-indexed column (no per-row O(n) recompute).

<b>Breaking:</b>
• <code>navigate_expand</code> gains a 5th arg (<code>relation_types</code>, defaults <code>NULL</code>). Positional callers using args 1–4 are unaffected; the old 4-arg overload is replaced.

<b>Upgrade:</b>
<pre>ALTER EXTENSION pgmnemo UPDATE TO '0.9.1'</pre>
