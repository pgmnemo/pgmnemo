**pgmnemo 0.12.0** — Typed Write API

Добавлены три новые функции для структурированной записи памяти агента:

• `remember_fact(entity_key, property, value, …)` — записывает именованный факт об объекте. Автоматически маршрутизирует статус: контактные данные человека (`email`, `phone`, `address`, `telegram`, `full_name`) всегда попадают в `candidate`; системные записи — в `validated`. Поддерживает слияние при повторной записи того же значения и битемпоральное вытеснение при изменении.

• `remember_event(entity_key, label, body, occurred_at, …)` — фиксирует событие с временной меткой. Неизменяем после записи.

• `remember_relation(from_key, to_key, relation_type, …)` — добавляет направленное именованное отношение. Идемпотентен для одного тройного ключа.

Вспомогательные функции: `canonical_slug()` — нормализует произвольный текст в канонический идентификатор (`person:ada_lovelace`); `_has_contact_pii()` — определяет PII-свойства; `guard_no_test_project()` — защищает продакшн от тестовых данных.

Все три функции работают напрямую в Postgres — без внешних вызовов, без двойной записи. Схема данных и API отзыва (`recall_fast`, `recall_lessons`, `recall_hybrid`) не изменились.

Typed writes — это паритет с существующими решениями (Mem0, Zep, Letta). Дифференциатор pgmnemo — единый SQL-план внутри Postgres, а не структурированная запись сама по себе.

Обновление: `ALTER EXTENSION pgmnemo UPDATE TO '0.12.0';`
