<b>pgmnemo v0.9.6</b> — типизация уроков, DAG-изоляция recall, лог миграции

Три запроса от сообщества в одном релизе.

<b>1. Типы контента в agent_lesson</b>

Новая колонка <code>item_kind</code> — теперь урок знает, что он такое: <code>note</code>, <code>skill_md</code>, <code>template</code>, <code>script</code>, <code>reference</code>, <code>config</code>, <code>spec</code>. По умолчанию — <code>note</code>, поведение существующих уроков не меняется.

Плюс <code>version_n</code> и <code>patch_count</code> для отслеживания редакций: когда скилл существенно переписан — инкрементируйте <code>version_n</code>, мелкая правка — <code>patch_count</code>.

<b>2. Workflow не recall'ит свои же выходы</b>

Новый параметр <code>exclude_dag_id</code> в <code>recall_hybrid()</code> и <code>recall_lessons()</code>. Передайте ID текущего workflow — и он не найдёт уроки, которые сам же породил в этом запуске. Позиционные вызовы без параметра работают как прежде.

<b>3. Лог миграции из других хранилищ</b>

Таблица <code>pgmnemo.memory_ingest_log</code> для операторов, которые переносят данные из старых memory-схем в pgmnemo. Фиксирует батчи: откуда, какие id, когда. После завершения миграции — просто дропается.

<b>Обновление (не-деструктивное):</b>
<pre>ALTER EXTENSION pgmnemo UPDATE TO '0.9.6'</pre>

Четыре новых колонки с DEFAULT-значениями, старые подписи функций не сломаны.
