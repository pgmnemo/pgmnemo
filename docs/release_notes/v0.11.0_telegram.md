<b>pgmnemo v0.11.0</b> — типизированный recall: можно искать память по типу записи

Теперь у <code>recall_hybrid()</code> есть необязательный фильтр по типу контента. Можно попросить вернуть только нужные виды памяти — например, только решения или только факты, — не перемешивая всё подряд в одной выдаче.

<b>Как это выглядит</b>

<pre>-- как раньше: вернёт всё
SELECT * FROM recall_hybrid('как мы решили делать деплой', ...);

-- новое: только записи нужных типов
SELECT * FROM recall_hybrid('как мы решили делать деплой', ..., p_content_types =&gt; ARRAY['decision','fact']);</pre>

<b>Полностью обратно совместимо</b>

• Новый параметр <code>p_content_types</code> — десятый, со значением по умолчанию.
• Старые вызовы (9 аргументов) работают как прежде, менять ничего не нужно.
• Если фильтр не передан — поведение в точности прежнее.

<b>Честно о том, что это и что это не</b>

Это удобный фильтр: он управляет <i>тем, что</i> достаётся из памяти, а не качеством ранжирования. Мы не обещаем прироста точности recall — формула ранжирования не менялась. Типизированный доступ к памяти — нужная базовая возможность (она есть и у соседей по рынку); этот релиз честно её добавляет.

<b>Обновление</b>

<pre>ALTER EXTENSION pgmnemo UPDATE TO '0.11.0';</pre>

MCP-сервер (если используется):
<pre>pip install --upgrade pgmnemo-mcp==0.11.0</pre>
