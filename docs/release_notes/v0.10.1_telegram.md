<b>pgmnemo v0.10.1</b> — hotfix: recall_fast NULL guard + recall_hybrid стабильность BM25

<b>Issue #84 — recall_fast: NULL embedding теперь выбрасывает исключение</b>

До v0.10.1 вызов <code>recall_fast(NULL::vector, ...)</code> возвращал строки с <code>score = NULL</code> — тихое повреждение данных вместо ошибки. Теперь выбрасывается <code>EXCEPTION P0001</code> с понятным сообщением:

<code>recall_fast: query_embedding IS NULL — use recall_hybrid() if you have query_text but no embedding.</code>

<b>Issue #87 — recall_hybrid: 4 исправления BM25-пути</b>

На корпусах с длинными/кириллическими запросами <code>recall_hybrid()</code> таймаутился до 20% вызовов с p95~910ms. Исправлено:

• Запрос обрезается до 200 символов перед <code>websearch_to_tsquery</code>
• Используется индексированный <code>full_text</code> (GIN) вместо per-row <code>to_tsvector(topic)</code>
• BM25-блок изолирован в <code>BEGIN … EXCEPTION WHEN query_canceled</code> — таймаут = graceful fallback на vector-only, а не ошибка
• <code>tsconfig</code> изменён на <code>'simple'</code> — корректная токенизация для кириллицы и структурированных текстов

Результат: 0% таймаутов, p95 = 44ms (было 910ms, улучшение в 20×).

<b>Обновление</b>

<pre>ALTER EXTENSION pgmnemo UPDATE TO '0.10.1';</pre>

⚠️ Миграция меняет <code>GENERATED ALWAYS AS</code> выражения для <code>topic_tsv</code>, <code>lesson_tsv</code>, <code>full_text</code> (английский стеммер → простой). GIN-индексы перестраиваются автоматически при UPDATE — на больших корпусах (&gt;100k уроков) запланируйте кратковременный рост I/O.

Новый GUC: <code>pgmnemo.bm25_budget_ms</code> (INT, default 250) — лимит BM25 в миллисекундах.

MCP-сервер (если используется):
<pre>pip install --upgrade pgmnemo-mcp==0.10.1</pre>
