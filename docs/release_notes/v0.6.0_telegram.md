🚀 <b>pgmnemo v0.6.0</b>

<b>1. ghost_count в pgmnemo.stats()</b>
<code>SELECT ghost_count FROM pgmnemo.stats();</code>
BIGINT — число активных уроков с <code>verified_at IS NULL</code> (нет commit_sha, нет artifact_hash). Скан: <code>agent_lesson WHERE is_active=TRUE AND verified_at IS NULL</code>.

<b>2. RAISE NOTICE при дедупликации в pgmnemo.ingest()</b>
Срабатывает когда <code>(role, topic, commit_sha)</code> совпадает с существующей строкой: старая закрывается (<code>t_valid_to = NOW()</code>), вставляется новая. В лог:
<code>NOTICE: content_hash_dedup_fired lesson_id=&lt;N&gt;</code>
psycopg2: <code>connection.notices</code>. Раньше ветка была молчаливой.

<b>3. pgmnemo.recall_stats</b>
<code>SELECT * FROM pgmnemo.recall_stats;</code>
Колонки: <code>function_name, calls, total_time, self_time</code>. Обёртка над <code>pg_stat_user_functions</code> с фильтром по схеме pgmnemo (<code>recall_lessons</code>, <code>recall_hybrid</code>, <code>ingest</code>). Требует <code>track_functions = 'pl'</code> или <code>'all'</code> в <code>postgresql.conf</code>.

<b>4. PostGIS spatial pre-filter</b> (<a href="https://github.com/pgmnemo/pgmnemo/blob/v0.6.0/docs/cookbook/postgis_spatial_recall.md">cookbook</a>)
<code>ST_DWithin</code> фильтрует кандидатов по координатам → IDs уходят в <code>restrict_lesson_ids</code>:
<code>SELECT pgmnemo.recall_hybrid(q, restrict_lesson_ids =&gt; ARRAY(SELECT id FROM geo WHERE ST_DWithin(geom, pt, 500)));</code>
Кейс: агент с гео-памятью (логистика, field-service). Паттерн переиспользуем для любого до-фильтра.

<b>5. MIGRATION.md §Rollback + USAGE.md §Tuning</b>
PostgreSQL не поддерживает downgrade расширений. Бэкап до <code>ALTER EXTENSION pgmnemo UPDATE</code>:
<code>COPY pgmnemo.agent_lesson TO '/tmp/backup.csv' CSV;</code>
Recency: <code>recency_factor = exp(-recency_weight × temporal_boost × age_days / 90)</code>. При дефолтах записи старше года → factor &lt; 0.01, исторический корпус выпадает молча. Таблица: <code>USAGE.md §Tuning</code>.
