**pgmnemo 0.12.1** — чистка продукта

Небольшой релиз без новых возможностей: убрали из кода продукта предположения, специфичные для конкретного потребителя.

• Защитная функция `guard_no_test_project()` больше не зашивает порог «production = id ≤ 100». Теперь порог настраивается самим потребителем: `SET pgmnemo.test_project_floor = N;` (по умолчанию выключено). Продукт больше не навязывает свою схему нумерации проектов.

• Комментарии в коде теперь ссылаются на собственную спецификацию продукта, а не на внутренние документы.

Поведение записи (`remember_fact` / `remember_event` / `remember_relation`) и отзыва не изменилось. Обновление: `ALTER EXTENSION pgmnemo UPDATE TO '0.12.1';`
