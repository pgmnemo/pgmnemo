<b>pgmnemo v0.9.5</b> — recall-recency signals + corpus curation 🧹

Корпус из 6 584 уроков, 92% из которых никогда не воспроизводились — это балласт в recall@k. Раньше нельзя было понять, когда урок последний раз был нужен: колонки `last_recalled_at` просто не существовало. Теперь она есть.

<b>Что изменилось:</b>

• <code>last_recalled_at</code> и <code>recall_count</code> — новые колонки в <code>agent_lesson</code>. Обновляются автоматически при каждом вызове recall-функций (<code>recall_hybrid</code>, <code>recall_lessons</code>, <code>navigate_locate</code>, <code>navigate_expand</code>).

• GUC <code>pgmnemo.track_recall_recency</code> (по умолчанию <code>on</code>) — выключите для сессионного benchmarking'а, где штамп не нужен.

• <code>pgmnemo.mark_stale()</code> — примитив чистки корпуса. По умолчанию работает в режиме <code>dry_run=TRUE</code> (только показывает кандидатов, ничего не трогает). Защищённые уроки (высокий confidence / importance=5 / с провенансом) никогда не удаляются автоматически.

```sql
-- Превью (всегда сначала):
SELECT * FROM pgmnemo.mark_stale() WHERE would_deprecate;

-- Применить после проверки:
SELECT COUNT(*) FROM pgmnemo.mark_stale(p_dry_run=>FALSE, p_cap=>200)
WHERE would_deprecate;
```

<b>Обновление (не-деструктивное):</b>
<pre>ALTER EXTENSION pgmnemo UPDATE TO '0.9.5'</pre>

Добавляются две колонки с DEFAULT NULL/0 — существующие строки не затронуты. Сигнатуры recall-функций не изменились.
