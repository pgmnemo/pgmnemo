**pgmnemo v0.6.3** — hotfix release (2026-05-24)

---

**`recall_lessons()` is now callable.** If you upgraded to v0.6.2 and hit
`psycopg2.errors.AmbiguousColumn: column reference "role" is ambiguous` on your first
recall call, this release fixes it.

**Root cause:** PL/pgSQL's name resolver saw `role` as ambiguous between the
`RETURNS TABLE (role TEXT, ...)` OUT variable and the `agent_lesson.role` column —
even when table-qualified. Fix: `#variable_conflict use_column` directive added to
both `recall_lessons()` and `recall_hybrid()`. Compile-time only — no change to
query plans, index selection, scoring formulas, or output values.

**Upgrade from v0.6.2:**
```sql
ALTER EXTENSION pgmnemo UPDATE TO '0.6.3';
```

**Fresh install:**
```bash
make install && psql -c "CREATE EXTENSION pgmnemo VERSION '0.6.3';"
```

---

**What else ships:**

📖 `pgmnemo.include_unverified` docs — the GUC is a **read filter only** (widens
recall to include lessons where `verified_at IS NULL`). It has no effect on the INSERT
provenance gate. `pgmnemo.gate_strict` controls writes; `include_unverified` controls
reads. These are now documented separately in `docs/USAGE.md`.

📖 Hybrid mode activation — explicit docs on the 3 conditions (disable_hybrid off,
query_text non-null, query_embedding non-null). **No corpus-size threshold exists** —
hybrid fires for corpora of any size. Includes SQL probe for `lesson_tsv` coverage and
backfill command.

📖 psycopg2 calling convention — working example with named `=>` parameter syntax,
`format_vector()` helper, and explanation of why embeddings must be passed as strings
with explicit `::vector` cast.
