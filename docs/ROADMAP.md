# Roadmap

**Status:** public-facing near-term roadmap  
**Last updated:** 2026-05-09

## Near-term priorities

### 1. Release coherence

- keep releases, docs, tags, and version metadata synchronized
- tighten install and upgrade documentation

### 2. Trust scaffolding

- security policy
- code of conduct
- issue templates
- PR template
- compatibility and production-readiness pages

### 3. Proof and benchmarks

- keep benchmark methodology public and reproducible
- document what is measured vs inferred
- make migration and adoption paths easier to evaluate

### 4. Memory-core improvements

- continued work on recall quality
- continued work on graph and provenance-related memory behavior

## How to read this roadmap

- shipped truth lives in `CHANGELOG.md`
- current API truth lives in `docs/USAGE.md`
- operator truth lives in `INSTALL.md`

This roadmap is directional, not a release contract.
